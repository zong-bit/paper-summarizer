## 1. 核心算法概述

### 1.1 整体排盘流程

```
用户输入 (阳历/农历 + 出生地经纬度)
    │
    ▼
① 真太阳时校正 ────────────────────────────── 核心前置步骤
    │
    ▼
② 年柱计算 ── 立春为界
    │
    ▼
③ 月柱计算 ── 节气为界 (12个"节")
    │
    ▼
④ 日柱计算 ── 儒略日/蔡勒公式
    │
    ▼
⑤ 时柱计算 ── 五鼠遁法
    │
    ▼
⑥ 扩展计算 ── 藏干/十神/纳音/神煞/大运/流年
```

### 1.2 关键算法详解

#### 1.2.1 真太阳时校正 (True Solar Time)

**为什么需要**：八字排盘以太阳位置为基准，而用户提供的出生时间通常是标准时区时间（如北京时间 UTC+8）。必须校正：

1. **经度差修正**：出生地经度与标准时区中心经度的差值
   - 公式：`经度差分钟 = (出生地经度 - 时区标准经度) × 4 分钟/度`
   - 例：广州 (113.27°E)，时区标准经度 120°E → `(113.27 - 120) × 4 = -26.92 分钟`

2. **均时差 (Equation of Time)**：地球椭圆轨道 + 地轴倾斜导致的太阳时差
   - 范围：约 -14 分钟 到 +16 分钟
   - 计算公式（基于日积角 `n = 儒略日 - 2451545.0`）：
   ```
   E = 229.18 × (0.000075 + 0.01868×cos(ω) - 0.03208×sin(ω)
                  - 0.01461×cos(2ω) - 0.00458×sin(2ω))
   其中 ω = 2π/365 × (n - 81)
   ```

3. **夏令时修正**：根据出生地是否实行 DST 自动判断

```typescript
interface SolarTimeResult {
  adjustedHour: number;    // 校正后的真太阳时 (0-24)
  adjustedMinute: number;
  longitudeDiff: number;   // 经度差 (分钟)
  equationOfTime: number;  // 均时差 (分钟)
  dst: boolean;            // 是否夏令时
}
```

#### 1.2.2 年柱计算

**规则**：以**立春**为年度分界，而非农历正月初一或公历1月1日。

- 天干 = `(年 - 3) % 10`，对应：甲1 乙2 丙3 丁4 戊5 己6 庚7 辛8 壬9 癸10
- 地支 = `(年 - 3) % 12`，对应：子1 丑2 寅3 卯4 辰5 巳6 午7 未8 申9 酉10 戌11 亥12
- **关键**：立春之前出生的，年柱仍为上一年的干支

```
立春 ≈ 公历 2月4日左右（精确到分钟，需查节气表）
```

#### 1.2.3 月柱计算

**规则**：以**12个"节"**（非中气）为月度分界：

| 月 | 节 | 太阳黄经 | 公历约期 |
|----|------|----------|----------|
| 寅月 | 立春 | 315° | 2月3-5日 |
| 卯月 | 惊蛰 | 345° | 3月5-7日 |
| 辰月 | 清明 | 15° | 4月4-6日 |
| 巳月 | 立夏 | 45° | 5月5-7日 |
| 午月 | 芒种 | 75° | 6月5-7日 |
| 未月 | 立秋 | 135° | 8月7-9日 |
| 申月 | 白露 | 165° | 9月7-9日 |
| 酉月 | 秋分 | 195° | 9月22-24日 |
| 戌月 | 寒露 | 225° | 10月8-9日 |
| 亥月 | 立冬 | 255° | 11月7-8日 |
| 子月 | 大雪 | 285° | 12月6-8日 |
| 丑月 | 小寒 | 315° | 1月5-7日 |

**月干推算（五虎遁法）**：

| 年干 | 寅月天干 | 口诀 |
|------|---------|------|
| 甲/己 | 丙寅 | 甲己之年丙作首 |
| 乙/庚 | 戊寅 | 乙庚之岁戊为头 |
| 丙/辛 | 庚寅 | 丙辛之年从庚起 |
| 丁/壬 | 壬寅 | 丁壬壬位顺行寻 |
| 戊/癸 | 甲寅 | 戊癸之年甲是初 |

月支固定：立春后为寅月，惊蛰后为卯月，依此类推。

#### 1.2.4 日柱计算

**方法**：基于儒略日 (Julian Day) 的偏移计算。

- 以已知参考日（如公元4年1月31日 = 甲子日）为基准
- 计算目标日期与参考日的儒略日差值
- 日干 = `(儒略日差值) % 10`，日支 = `(儒略日差值) % 12`

```typescript
// 儒略日公式 (Gregorian calendar)
function toJulianDay(year: number, month: number, day: number): number {
  const a = Math.floor((14 - month) / 12);
  const y = year + 4800 - a;
  const m = month + 12 * a - 3;
  return day + Math.floor((153 * m + 2) / 5) + 365 * y
       + Math.floor(y / 4) - Math.floor(y / 100) + Math.floor(y / 400) - 32045;
}
```

**注意**：日柱在子时 (23:00) 换日（不分早晚子时的流派），或凌晨 00:00 换日（分早晚子时流派）。

#### 1.2.5 时柱计算

**规则**：真太阳时校正后，按地支时辰表：

| 时辰 | 时间范围 | 地支 |
|------|---------|------|
| 子时 | 23:00-01:00 | 子 |
| 丑时 | 01:00-03:00 | 丑 |
| 寅时 | 03:00-05:00 | 寅 |
| 卯时 | 05:00-07:00 | 卯 |
| 辰时 | 07:00-09:00 | 辰 |
| 巳时 | 09:00-11:00 | 巳 |
| 午时 | 11:00-13:00 | 午 |
| 未时 | 13:00-15:00 | 未 |
| 申时 | 15:00-17:00 | 申 |
| 酉时 | 17:00-19:00 | 酉 |
| 戌时 | 19:00-21:00 | 戌 |
| 亥时 | 21:00-23:00 | 亥 |

**时干推算（五鼠遁法）**：

| 日干 | 子时天干 | 口诀 |
|------|---------|------|
| 甲/己 | 甲子 | 甲己还加甲 |
| 乙/庚 | 丙子 | 乙庚丙作初 |
| 丙/辛 | 戊子 | 丙辛从戊起 |
| 丁/壬 | 庚子 | 丁壬庚子居 |
| 戊/癸 | 壬子 | 戊癸何方发，壬子是真途 |

### 1.3 扩展计算

| 项目 | 算法 |
|------|------|
| **地支藏干** | 固定映射表（如寅→甲丙戊） |

| **流年** | 从大运起始年逐年推算 |
| **刑冲合害** | 地支关系表（六冲、三合、六合、三刑、自刑等 15+ 种） |

### 2.3 技术选型建议

> **推荐方案：基于 tyme4ts + 自研八字排盘规则引擎**

**理由**：

1. **tyme4ts** 是 6tail 团队维护的专业天文历法库，提供：
   - 精确的 24 节气计算（基于太阳黄经）
   - 干支历法转换
   - 真太阳时计算（均时差公式）
   - 农历/公历互转
   - 无第三方依赖，纯 TypeScript

2. **自研规则引擎** 的优势：
   - 完全掌控业务逻辑（早/晚子时切换、流派差异等）
   - 可按需扩展（紫微斗数、奇门遁甲等后续模块）
   - 无外部依赖风险
   - 可针对 ez-mystic 的业务场景优化

3. **不推荐直接依赖 mystilight-8char 或 bazi.js**：
   - 均无真太阳时支持
   - 维护活跃度低
   - 功能覆盖面不足

4. **可参考 shunshi-bazi-core 的实现**：
   - 其架构设计（`bazi.ts` / `relations.ts` / `shensha.ts` / `solarTime.ts`）值得借鉴
   - 但无需完全依赖，可独立实现

### 3.4 核心算法流程

```
排盘入口 (calculateBazi(input))
│
├─▶ Step 1: 日期解析
│     若 lunar → 用 tyme4ts 转为 solar
│     校验日期范围 (1900-2100 建议范围)
│
├─▶ Step 2: 真太阳时校正 (可选)
│     经度差修正 = (longitude - 120) × 4 min
│     均时差 = EquationOfTime(year, month, day)
│     夏令时修正 = checkDST(year, month, day, timezone)
│     adjustedTime = originalTime + 经度差 + 均时差 + DST修正
│     → 可能跨日，需更新 date
│
├─▶ Step 3: 年柱计算
│     查当年节气表，获取立春精确时刻
│     if (birthTime < 立春时刻) → 年柱 = 上一年
│     else → 年柱 = 当年
│     年干 = (year - 3) % 10
│     年支 = (year - 3) % 12
│
├─▶ Step 4: 月柱计算
│     查 birthTime 落在哪个"节"之后
│     月支 = 对应地支 (立春→寅, 惊蛰→卯, ...)
│     月干 = 五虎遁(年干) → 寅月天干
│     顺推至对应月份
│
├─▶ Step 5: 日柱计算
│     jdn = JulianDayNumber(year, month, day)
│     dayGan = (jdn - 参考日JDN) % 10
│     dayZhi = (jdn - 参考日JDN) % 12
│     子时分界处理 (sect 1: 23:00换日, sect 2: 00:00换日)
│
├─▶ Step 6: 时柱计算
│     hourBranch = 时辰表(adjustedTime)
│     hourGan = 五鼠遁(日干) → 子时天干
│     顺推至对应时辰
│
├─▶ Step 7: 扩展计算
│     藏干 → 固定表查询
│     十神 → 日干 vs 各干比较
│     纳音 → 60甲子查表
│     五行 → 统计各柱五行
│     神煞 → 年支/日支查表
│     大运 → 起运年龄 = 距节气天数 ÷ 3
│     刑冲合会 → 地支关系矩阵
│
└─▶ Step 8: 构建输出
      组装 BaziChart 对象
      JSON 序列化
```

## 4. 关键技术决策

### 4.1 真太阳时精度

| 方案 | 精度 | 复杂度 | 推荐度 |
|------|------|--------|--------|
| 仅经度修正 | ~±2 分钟 | 低 | ⭐⭐ |
| 经度 + 均时差 | ~±30 秒 | 中 | ⭐⭐⭐⭐⭐ |
| 经度 + 均时差 + DST | ~±30 秒 | 中高 | ⭐⭐⭐⭐⭐ |

**建议**：默认启用完整真太阳时校正，提供配置项可降级为仅经度修正。

### 4.2 节气数据来源

| 方案 | 精度 | 数据量 | 推荐度 |
|------|------|--------|--------|
| 硬编码查表 (年→节气时刻) | 秒级 | ~200年×24 = 4800条 | ⭐⭐⭐⭐ |
| 天文公式实时计算 | 秒级 | 零数据 | ⭐⭐⭐⭐⭐ |
| 依赖 tyme4ts | 秒级 | 零数据 | ⭐⭐⭐⭐⭐ |

**建议**：使用 tyme4ts 的 `getJieQi()` API，避免维护天文数据。

### 4.3 子时处理策略

| 流派 | 规则 | 适用场景 |
|------|------|----------|
| 不分早晚子时 (sect 1) | 23:00 换日柱 | 主流，推荐默认 |
| 分早晚子时 (sect 2) | 00:00 换日柱 | 部分传统流派 |

**建议**：提供 `sect` 配置选项，默认 `1`（不分早晚子时），文档中说明差异。

### 4.4 城市经纬度数据

- 内置 300+ 中国主要城市经纬度表（约 15KB）
- 支持用户手动输入经纬度
- 可选：集成 `geoip-lite` 根据 IP 估算（仅服务端）

## 5. 与 tyme4ts 的集成

```
┌─────────────┐     ┌──────────────────────────┐
│  八字规则引擎  │────▶│     tyme4ts (天文历法)     │
│              │     │                          │
│  四柱计算     │     │  getJieQi(year)          │
│  真太阳时     │     │  getEquationOfTime(date) │
│  扩展计算     │◀────│  solar2lunar()           │
│              │     │  lunar2solar()           │
│              │     │  ganZhi 计算              │
│              │     │  农历置闰                  │
└─────────────┘     └──────────────────────────┘
```

**tyme4ts 提供**：
- 24 节气精确时刻（太阳黄经算法）
- 均时差计算
- 公历 ↔ 农历转换
- 干支历法基础计算

**自研引擎提供**：
- 八字排盘业务逻辑（年柱立春判断、月柱节气判断等）
- 真太阳时校正流程
- 五虎遁/五鼠遁
- 藏干/十神/纳音/神煞/大运/刑冲合会
- 命盘构建与输出
